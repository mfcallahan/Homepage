Software Development

My main area of interest in programming is desktop and back end web development using the Microsoft .NET framework. C# is my language of choice when I need to develop a custom solution. It is a powerful language with a robust base class library and awesome features like LINQ, dependency injection, asynchronous programming, and garbage collection.

Sometimes I just don't need the overhead of the massive .NET framework, and Python is another favorite language. It is the perfect language for me to automate repetitive tasks or process large batches of data. Python is also what I strongly recommend as a first language to learn for programming beginners.

I am also an Angular developer, and have made this my preferred front end framework for developing web applications. The provided command line tools, integrated testing framework, and being built around TypeScript make this a framework with which I greatly enjoy working.

  • ArcGIS API Unit Test Demo
  • App link
  • Blog Post
  • Source code
When using the esri-loader and the ArcGIS API for JavaScript inside an Angular application, I ran into an issue writing unit tests: the API modules are lazy loaded at run time and HTTP requests are made to fetch the resources, but I needed to mock this behavior in tests. This app demonstrates an approach for testing code which has a dependency on the esri-loader. Using dependency injection and the Facade pattern, an abstraction for the methods which load the API modules can be created. This allows the esri-loader dependency to be more easily mocked, and eliminates actual HTTP requests being made in a unit test.
  • ArcGIS API ES Modules
  • Source code
Beginning with version 4.18 of the API, Esri has begun offering the ArcGIS API for JavaScript via standard ES Modules which can be directly imported into classes, no longer necessitating the use of the esri-loader or the arcgis-webpack-plugin to load the Dojo-based API modules. This repository contains an Angular app to experiment with the new ArcGIS ES modules in preparation for migrating my applications away from the esri-loader.
